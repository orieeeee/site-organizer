사이트 정리 페이지 사용법

index.html과 같은 폴더의 sites.json만 수정하면 자동으로 목록이 갱신됩니다.

1) 구조

{
  "sites": [
    {
      "id": "고유아이디",
      "name": "표시 이름",
      "url": "https://example.com",
      "desc": "설명",
      "category": "카테고리",
      "tags": ["태그1","태그2"],
      "added": 1730793600
    }
  ]
}

2) 편집

  • id: 중복되지 않는 문자열 (핀/즐겨찾기 저장에 사용)
  • added: 최근추가순 정렬용 유닉스 타임스탬프 (생략 가능)

3) 기능

  • 검색: 이름/설명/URL/태그 전체 검색
  • 카테고리 선택, 태그 클릭 필터
  • 정렬: 핀(★) 우선 > 이름, 이름순, 최근추가순
  • 핀(★): 로컬 저장 (브라우저별로 유지)
  • 다크모드 토글
  • 파비콘 자동 표시

4) 배포 (GitHub Pages)

  1. 이 폴더를 리포지토리로 올립니다. (예: site-organizer)
  2. 리포 설정 > Pages > Branch: main / root 선택
  3. URL이 생성되면 접속하여 확인합니다.
  4. sites.json만 업데이트하면 바로 반영됩니다 (캐시가 남으면 새로고침).

5) Google Sheet로 관리하고 싶다면?

시트를 웹에 공개 -> CSV로 게시 후 URL을 받아 fetch로 불러오고, CSV를 파싱해 위 JSON 포맷으로 변환하면 됩니다. 필요하면 그 버전도 만들어 드릴게요.
